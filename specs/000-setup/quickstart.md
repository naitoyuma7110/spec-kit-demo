# Quickstart Guide: プロジェクトセットアップ検証

**Date**: 2026-02-12
**Feature**: 000-setup

## 目的

セットアップが正常に完了したことを検証するための手順。

## 前提条件

- Node.js v20がインストールされている
- Voltaがインストールされている（オプション）
- PostgreSQLがインストールされ、起動している
- プロジェクトの依存関係がインストールされている

## 検証手順

### 1. 開発サーバー起動確認

#### フロントエンド

```bash
cd frontend
yarn dev
```

**期待される結果**: 
- サーバーが http://localhost:3000 で起動
- ブラウザでアクセス可能
- エラーが発生しない

#### バックエンド

```bash
cd backend
yarn start:dev
```

**期待される結果**: 
- サーバーが http://localhost:3001 で起動
- エラーが発生しない

### 2. データベース接続テスト

```bash
cd backend
yarn prisma db push
yarn prisma generate
```

**期待される結果**: 
- データベーススキーマが正常に適用される
- Prismaクライアントが生成される
- エラーが発生しない

### 3. 環境変数の確認

`.env`ファイルが存在し、以下の環境変数が設定されていることを確認：

- `DATABASE_URL`: データベース接続文字列
- `JWT_SECRET`: JWT署名用の秘密鍵
- `NODE_ENV`: 環境設定（開発環境では`development`）

### 4. セットアップ検証スクリプト（オプション）

セットアップ検証を自動化するスクリプトを作成する場合：

```bash
# 検証スクリプトの実行
yarn verify-setup
```

**期待される結果**: 
- すべての検証が成功
- エラーが発生しない

## トラブルシューティング

### ポートが既に使用されている場合

- フロントエンド: `nuxt.config.ts`でポートを変更
- バックエンド: `main.ts`でポートを変更

### データベース接続エラー

- PostgreSQLが起動しているか確認
- `.env`の`DATABASE_URL`が正しいか確認
- データベース`todo_app_dev`が作成されているか確認

### 依存関係のインストールエラー

- `yarn install`を再実行
- `node_modules`を削除して再インストール
- ネットワーク接続を確認

## 成功基準

すべての検証手順が成功し、以下の条件を満たすこと：

- ✅ フロントエンド開発サーバーが正常に起動
- ✅ バックエンド開発サーバーが正常に起動
- ✅ データベース接続が正常
- ✅ 環境変数が正しく設定されている
- ✅ エラーが発生しない
